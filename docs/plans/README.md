# Planning Folder

This folder organizes planning into separate focused documents.

## Files

1. `DEVELOPMENT_PLAN.md`
   - Active product-first development roadmap, weekly plan, and daily checklist.
2. `DEPLOYMENT_PLAN.md`
   - Deferred deployment and release tracks, with links to detailed ops documents.
3. `OTHER_TASKS_PLAN.md`
   - Deferred and stretch tasks tracked separately from active development.

## Usage

1. Start with `DEVELOPMENT_PLAN.md` for current execution.
2. Use `OTHER_TASKS_PLAN.md` to track deferred/non-active work.
3. Reactivate `DEPLOYMENT_PLAN.md` only when deployment work is resumed.
